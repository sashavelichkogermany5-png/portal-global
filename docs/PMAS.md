# P-M-A-S — эволюционная система принятия решений

Этот документ описывает рабочий шаблон P-M-A-S для Portal Global и формат входных данных
для генератора отчётов `scripts/pmas-run.js`.

## Когда применять
- Есть больше одной реальной опции.
- Ошибка дорого стоит (auth, tenant, autopilot, финансы, релизы).
- Чувство и логика расходятся.

## Когда не применять
- Фактический вопрос без выбора.
- Единственный разумный вариант.
- Обратимое тривиальное решение.

## Структура (шаги)
1. Problem — одна строка.
2. Fitness — 3 измеримых критерия.
3. Varianten — 4 реально разные опции.
4. Fakten-check — только факты из системы.
5. Stress-test — 3 сценария (S1/S2/S3) с оценками.
6. Selektion — таблица и победитель.
7. Kreuzung — финальная “дочерняя” версия.

## Формат для автоматического отчёта
Файл `docs/pmas-cases.json` содержит массив кейсов. Каждый кейс:

- id: короткий идентификатор
- title: название
- problem: строка
- fitness: массив из 3 критериев
- variants: массив из 4 вариантов (A,B,C,D)
- facts: список фактов (строки)
- scenarios: S1/S2/S3 с оценками по F1/F2/F3
- selection: результат (winner, runnerUp)
- crossover: описание ребёнка
- result: вывод/рекомендация

Генерация отчёта:
```bash
node scripts/pmas-run.js
```
Отчёт будет записан в `docs/pmas-report.md`.
